[요약 앵커 가이드 — "상담내용요약" (전체상담) Task]

1. 요약 대상의 본질
   "상담내용요약"은 상담사가 통화 종료 후 업무 시스템에 기록하는 후처리 메모이다.
   고객의 인입 사유, 확인한 사항, 상담사가 안내·처리한 결과를 사실 중심으로 압축한다.
   핵심은 "이 통화에서 무엇이 확인되었고 어떤 결론이 났는가"이다.

2. 요약 구조
   "[고객 인입 사유/문의] + [확인 사항] + [안내·처리 결과·후속 조치]" 순서로 작성한다.
   - 포인트 단위로 나열한다. "/" 또는 줄바꿈으로 구분.
   - 상담사 관점의 업무 기록체를 사용한다.
     (예: "~안내함", "~처리함", "~확인", "~전달드림", "~콜백 생성")
   - 특히 "상담사가 안내/처리한 결과"가 가장 중요한 부분이다.
     고객 문의만 요약하고 상담사의 안내/결론을 빠뜨리면 오답이 된다.

3. 추출 우선순위 — STT에서 무엇을 뽑을 것인가
   ① 상담사가 최종 안내한 결론/결과 (가장 중요)
     → "~불가 안내", "~처리 완료", "~부서 전달", "~콜백 생성", "~연락 예정"
     → 이것이 빠지면 오답이다. 반드시 포함할 것.
   ② 구체적 수치 — 금액, 날짜, 건수, 증권번호, 전화번호
     → "4,482,670원", "10/21 만기", "계약건 3건", "1588-9898", "95만 8050원"
     → 숫자가 Reference 키워드에 포함되어 있으면 정확히 기재해야 한다. 생략하면 오답.
   ③ 고객의 인입 사유 및 핵심 요청
     → "해지환급금 문의", "보험금 청구", "카드 변경요청", "납입정지 요청"
   ④ 상담사가 확인한 조건/상태
     → "대출가능상품 아님", "미납 없음", "청약상태 철회 아님(반송 진행)"
     → 부정형 조건("~아님", "~불가", "~없음")은 특히 정확히 기재할 것
   ⑤ 이하 생략 대상: 본인확인 절차, ARS 안내 과정, 상담사의 인사말, 고객의 감정 표현

4. 금지 사항
   — 키워드 관련 —
   - 금액·날짜·건수·번호를 임의로 생략하거나 반올림하지 말 것
     ("이자 포함 200만3150원" → "대출금을 상환" ✗)
   - 부정형 결론을 긍정형으로 반전시키지 말 것
     ("청약상태 철회 아님" → "계약 철회를 요청함" ✗)
     ("가입건 확인되지 않음" → "가입 사실을 확인함" ✗)
   - 고유 용어를 자의적으로 바꾸지 말 것
     ("콜백 생성" → "연락처 전달" ✗, "AIG" → "AIA" ✗)

   — 과답 방지 —
   - "본인 확인 절차를 거친 후" 등 절차 서술을 추가하지 말 것 (과답의 가장 흔한 원인)
   - "상담을 종료함", "추가 요청 사항이 없음을 확인" 등 마무리 문구를 넣지 말 것
   - STT에 존재하더라도 Reference에 포함되지 않을 수준의 세부사항은 생략할 것
     (상품명 전체, 약관 조항, 상담사 개인정보, 고객의 감정 표현 등)

   — 문체 관련 —
   - "고객은 ~하였으며, 상담사는 ~하였다" 식의 장문 서술체를 피하고,
     "~문의 / ~안내함 / ~처리 완료" 식의 간결한 업무 기록체를 사용할 것
   - Reference 대비 3배 이상 장황하면 과답이 될 가능성이 높다

5. STT 오전사 대응
   STT는 음성 인식 결과이므로 오탈자, 문맥 단절, 의미 불명확한 표현이 있을 수 있다.
   - 문맥상 의미를 유추할 수 있으면 원래 의도에 맞게 자연스럽게 요약
   - 완전히 의미를 알 수 없는 부분은 생략
   - 보험 용어(약관대출, 납입면제, 해지환급금, 콜백 등)는 정확한 업계 표현으로 교정

6. 자주 틀리는 패턴과 교정

   [키워드 누락 — 오답 최다 원인]
   ✗ "대출 전액 상환을 요청하였으며 ... 상환 처리를 완료하였다"
   ✓ "대출 전액상환 요청 / 이자 포함 200만3150원 출금 완료"

   ✗ "고객은 청약을 반송하고 ... 계약 철회를 요청함"
   ✓ "가입건 취소요청 / 청약상태 철회 아닌 담당부서 반송 진행 안내 / 빠른 반송요청 전달"

   ✗ "카톡으로 보험료 미납 안내 문자를 받았다는 이유로 상담을 요청하였으며..."
   ✓ "카톡 보험료 미납 확인요청 / 납입면제건 포함 2건 미납 없음 안내 / 미납문자 아니며 10/21 만기 안내임"

   [과답 — 불필요 정보 추가]
   ✗ "고객의 본인 확인 절차를 거친 후, 계약 정보를 확인한 결과..."
   ✓ (본인확인 절차 생략, 확인 결과만 기재)

   ✗ "추가 요청 사항이 없음을 확인하고 상담을 종료함"
   ✓ (마무리 문구 생략)
