[요약 앵커 가이드 — "개선요청사항" 요약 Task]

1. 요약 대상의 본질
   "개선요청사항"은 고객이 통화 중 제시한 서비스 개선 방향·요구를 추출한 것이다.
   만족도 조사 후처리 양식의 "개선요청사항" 란에 기입할 내용이라고 생각하라.
   "불편사항"과 달리, 고객이 "어떻게 바뀌었으면 좋겠는지"에 초점을 맞춘다.
   상담사가 건의 진행을 약속한 내용도 포함 가능하다.

2. 요약 구조
   "[구체적 개선 요청 내용]" 형태의 간결한 문장으로 작성한다.
   - Reference 예시:
     "상담사와 직접 연결할 수 있는 시스템이 있으면 좋겠음"
     "불필요하게 고객에게 서류를 요구하는 일이 없도록 해야 함"
     "청약녹취 콜 간소화 할 수 있는지 건의진행하겠다 안내"
     "사람을 늘리라고하심"
     "잘고쳐줬으면 좋겠음"
   - 고객의 원어/구어체를 가능한 한 살린다.
   - 개선 요청이 없으면 상담사의 건의 약속("~건의진행하겠다 안내")을 기재한다.
   - 대화에서 개선 요청이 전혀 추출되지 않는 경우에도 "없음"이라 쓰지 말고,
     불편사항에서 파생된 자연스러운 개선 방향을 한 문장으로 기술한다.

3. 추출 우선순위 — STT에서 무엇을 뽑을 것인가
   ① 고객이 직접 제시한 개선 방향 (가장 중요)
     → "상담사와 직접 연결되면 좋겠다", "사람을 늘려라", "한번에 서류 다 말해달라"
   ② 상담사가 건의/전달을 약속한 내용
     → "건의진행하겠다", "개선 요청 전달드리겠다", "참고해서 개선하겠다"
   ③ 고객의 구체적 대안 제시
     → "24시간 뒤에 풀린다면 다른 방법을 찾을 수 있다", "타사처럼 설치없이 바로 채팅되면 좋겠다"
   ④ 이하 생략: 불편사항 재서술, 상담 경위, 보험계약 세부사항

4. 금지 사항
   — Reference가 있을 때 —
   - Reference의 키워드를 자의적으로 변형하지 말 것
     ("불필요한 서류요구 방지" → "수술 기록지로 청구 가능하도록" ✗)
   - Reference에 포함된 구체적 표현(구어체 포함)을 생략하지 말 것
     ("나이먹은사람 생각해서" → 생략 ✗)
     ("고객들한테 기분나쁜말 하지말라함" → "응대 개선 요청" ✗)
   - 상담사의 건의 안내가 Reference에 있으면 반드시 포함
     ("건의진행하겠다 안내" 누락 → 오답)

   — Reference가 없을 때 —
   - "없음"으로 생성하면 오답. STT에서 개선 요청을 반드시 추출할 것.
   - 불편사항을 그대로 반복하지 말고, "~하도록 개선 요청" 형태로 변환
   - 추출한 개선 요청은 한 가지 핵심에만 집중 (다수 나열 금지)

   — 과답 방지 —
   - Reference에 없는 추가 개선 요청을 STT에서 가져오지 말 것
   - 불편사항의 원인/배경을 개선요청사항에 포함하지 말 것
   - "고객이 ~를 강조하며" 등 서술적 표현 불필요

5. STT 오전사 대응
   STT는 음성 인식 결과이므로 오탈자, 문맥 단절이 있을 수 있다.
   - 문맥상 유추 가능하면 원래 의도에 맞게 자연스럽게 요약
   - 의미를 알 수 없는 부분은 생략
   - 고객의 구어체는 가능한 한 원형을 살릴 것

6. 자주 틀리는 패턴과 교정

   [키워드 변형 — 오답]
   ✗ "고객이 수술 기록지로도 보험금 청구가 가능하도록 개선 요청함"
   ✓ "불필요하게 고객에게 서류를 요구하는 일이 없도록 해야 함"

   ✗ "보장서류 심사 처리 기간 단축 요청"
   ✓ "빠르게 심사 진행될 수 있도록 건의진행하겠음"

   [구체적 표현 생략 — 오답]
   ✗ "사전에 고객에게 직접 확인 절차를 거치는 개선 요청을 함"
   ✓ "고객에게 사전에 고지를 해야하고 예를 들어 24시간 뒤에 풀린다던가 하면 고객은 다른 방법을 문의하고 방법을 찾아줄 것"

   ✗ "상담사의 말이 빠르다는 불편을 전달하고 개선하겠다고 약속함"
   ✓ "나이먹은사람 생각해서 말빠른건 개선되야함"

   ["없음" 생성 — 오답]
   ✗ "없음"
   ✓ STT에서 불편 상황의 개선 방향을 추출하여 한 문장으로 작성
