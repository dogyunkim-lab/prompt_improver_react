아래 Phase 1 분석 결과를 바탕으로 3개의 프롬프트 후보 구조를 결정하라.
이 단계에서는 프롬프트 텍스트를 작성하지 않고 구조만 설계한다.

[핵심 원칙 — 반드시 상기하라]
요약 Task: "{generation_task}"
이 Task의 요약은 범용적 요약이 아니다. 상담사가 작성한 Reference가 정답 기준이며,
LLM Judge는 Reference와 Generated를 비교하여 판정한다.
정답 판정을 받으려면 LLM이 Reference와 동일한 핵심 내용을 요약해야 한다.
프롬프트 설계의 핵심은 LLM이 요약 Task의 목적에 맞는 정보를 정확히 선별하여
Reference가 담고 있는 핵심 내용을 빠짐없이 요약하도록 유도하는 것이다.
문체나 종결어미 같은 표면적 형식이 아니라 "어떤 내용을 포함/생략하는가"에 집중하라.

[설계 제약]
- 사용 모델: Qwen3-30B-A3B-Instruct (Dify 워크플로우)
- 노드는 최대 3개까지 허용 (단일 노드도 가능)
- 각 노드는 독립된 프롬프트를 가짐
- reasoning(thinking) ON/OFF를 노드별로 지정 가능
- 단일 노드 구조: 노드 A만 사용
- 2노드 구조: 노드 A(추출/분석) → 노드 B(생성/검증)
- 3노드 구조: 노드 A(추출) → 노드 B(생성) → 노드 C(검증)

[출력 형식 - JSON만 출력]
{{
  "candidates": [
    {{
      "label": "A",
      "node_count": 2,
      "node_roles": ["STT에서 핵심 불편사항 추출", "추출된 내용으로 요약 생성"],
      "node_reasoning_config": [true, false],
      "node_io": [
        {{"input": ["stt"], "output": "extracted_info"}},
        {{"input": ["stt", "extracted_info"], "output": "generated"}}
      ],
      "rationale": "이 구조를 선택한 근거 (2~3문장)",
      "focus_patterns": ["이 후보가 집중적으로 해결하려는 error_pattern 키워드 목록"]
    }},
    {{
      "label": "B",
      "node_count": 1,
      "node_roles": ["단일 노드로 추출+생성+검증 통합"],
      "node_reasoning_config": [true],
      "node_io": [
        {{"input": ["stt"], "output": "generated"}}
      ],
      "rationale": "...",
      "focus_patterns": ["..."]
    }},
    {{
      "label": "C",
      "node_count": 3,
      "node_roles": ["STT 정보 추출", "불편사항 요약 생성", "누락/환각 검증"],
      "node_reasoning_config": [true, false, true],
      "node_io": [
        {{"input": ["stt"], "output": "extracted_info"}},
        {{"input": ["stt", "extracted_info"], "output": "draft"}},
        {{"input": ["stt", "draft"], "output": "generated"}}
      ],
      "rationale": "...",
      "focus_patterns": ["..."]
    }}
  ],
  "design_summary": "전체 설계 전략 요약 (learning rate 반영)"
}}

주의사항:
- 반드시 3개 후보를 생성하라.
- node_roles, node_reasoning_config, node_io의 배열 길이는 반드시 node_count와 일치해야 한다.
- node_io: 각 노드의 입력 변수(input)와 출력 변수(output)를 명시하라.
  첫 번째 노드의 input에는 반드시 "stt"를 포함하고, 마지막 노드의 output은 반드시 "generated"여야 한다.
- focus_patterns는 Phase 1의 error_pattern_ranking에 있는 패턴 키워드를 참조하라.
- JSON 외에 다른 텍스트를 출력하지 마라.
